📚 Classes & Objects (Kelas & Objek)

Teori: Class adalah blueprint atau cetak biru untuk membuat objek. Seperti denah rumah, kita bisa membuat banyak rumah dari denah yang sama.

Analogi Kehidupan Nyata:

  • 🏗️ Class = Denah rumah (template)
  • 🏠 Object = Rumah yang sudah dibangun (instance)
  • 📱 Class iPhone = Template, object iPhone saya = Ponsel nyata saya
  • 🎮 Class Character dalam game = Template, object Hero saya = Karakter saya
# Membuat class (blueprint)
class Siswa:
    def __init__(self, nama, nilai):
        self.nama = nama
        self.nilai = nilai

# Membuat object dari class (instance)
siswa1 = Siswa("Ari", 85)
siswa2 = Siswa("Budi", 90)

# Analogi: 
# Class Siswa = Denah rumah
# siswa1, siswa2 = Rumah yang sudah dibangun

🎨 Attributes & Methods (Atribut & Metode)

Teori: Attribute adalah data/sifat objek. Method adalah action/perilaku objek.

Analogi Kehidupan Nyata:

  • 🐕 Anjing = Class
  • 🏷️ Attribute = Warna, umur, nama (sifat-sifat anjing)
  • 🎬 Method = Menggonggong, berlari, makan (tindakan/perilaku)
class Anjing:
    def __init__(self, nama, warna):
        # Attributes (sifat/data)
        self.nama = nama
        self.warna = warna
    
    # Methods (tindakan/perilaku)
    def gonggong(self):
        print(f"{self.nama}: Guk guk!")
    
    def berlari(self):
        print(f"{self.nama}: Berlari-larian!")

# Menggunakan
anjing1 = Anjing("Buddy", "coklat")
anjing1.gonggong()     # Buddy: Guk guk!
anjing1.berlari()      # Buddy: Berlari-larian!

👨‍👧‍👦 Inheritance (Pewarisan)

Teori: Inheritance memungkinkan class baru mewarisi sifat dari class lama. Menghindari duplikasi kode dan membuat struktur hirarkis.

Analogi Kehidupan Nyata:

  • 👨 Ayah = Class induk (parent)
  • 👦 Anak = Class anak (child) yang mewarisi ciri-ciri ayah
  • 🐕 Hewan = Class induk, 🐕 Anjing = Class anak
# Class induk (Parent)
class Hewan:
    def __init__(self, nama):
        self.nama = nama
    
    def bergerak(self):
        print(f"{self.nama}: Bergerak...")

# Class anak (Child) - mewarisi dari Hewan
class Anjing(Hewan):
    def bergerak(self):
        print(f"{self.nama}: Berlari...")
    
    def gonggong(self):
        print(f"{self.nama}: Guk guk!")

# Menggunakan
anjing = Anjing("Buddy")
anjing.bergerak()    # Buddy: Berlari...
anjing.gonggong()    # Buddy: Guk guk!

🎭 Polymorphism (Banyak Bentuk)

Teori: Polymorphism berarti satu function/method yang sama bisa memiliki perilaku berbeda tergantung objeknya.

Analogi Kehidupan Nyata:

  • 🔊 Suara "buka" = Berbeda di gerbang, pintu rumah, email
  • 🐕🐈🦆 Metode "bersuara()" = Anjing gonggong, kucing meow, bebek kwak
  • 📱 Tombol "buka" di app = Buka file, folder, link
class Hewan:
    def bersuara(self):
        pass

class Anjing(Hewan):
    def bersuara(self):
        print("Guk guk!")

class Kucing(Hewan):
    def bersuara(self):
        print("Meow!")

class Bebek(Hewan):
    def bersuara(self):
        print("Kwak kwak!")

# Polymorphism - satu method, banyak perilaku
animals = [Anjing(), Kucing(), Bebek()]
for animal in animals:
    animal.bersuara()  # Setiap animal bersuara berbeda!

🔒 Encapsulation (Enkapsulasi)

Teori: Encapsulation adalah menyembunyikan detail internal dan hanya menampilkan interface yang diperlukan.

Analogi Kehidupan Nyata:

  • 📱 Ponsel = User hanya perlu klik tombol, tidak perlu tahu internal
  • 🚗 Mobil = Driver hanya perlu pijak pedal, tidak perlu tahu mesin
  • 🏦 ATM = User hanya input PIN, tidak bisa lihat rekening orang lain
class Rekening:
    def __init__(self, saldo_awal):
        # Private attribute (hanya bisa diakses dari dalam class)
        self.__saldo = saldo_awal
    
    # Public method (bisa diakses dari luar)
    def tarik_uang(self, jumlah):
        if jumlah <= self.__saldo:
            self.__saldo -= jumlah
            print(f"Saldo: {self.__saldo}")
        else:
            print("Saldo tidak cukup!")
    
    def lihat_saldo(self):
        return self.__saldo

# Menggunakan
rek = Rekening(1000)
rek.tarik_uang(300)        # ✓ Bisa
# rek.__saldo = 999999      # ✗ Error, tidak bisa akses private

✨ Special Methods (Metode Khusus)

Teori: Special methods adalah method dengan double underscore yang memberikan "super power" ke class. Contoh: __init__ untuk constructor, __str__ untuk representasi string.

Analogi Kehidupan Nyata:

  • 👶 __init__ = Proses lahir (inisialisasi)
  • 🗣️ __str__ = Bagaimana cara object mendeskripsikan diri
  • 🔢 __len__ = Panjang object
class Buku:
    def __init__(self, judul, penulis, halaman):
        self.judul = judul
        self.penulis = penulis
        self.halaman = halaman
    
    def __str__(self):
        """Representasi string object"""
        return f"{self.judul} oleh {self.penulis}"
    
    def __len__(self):
        """Panjang dari object"""
        return self.halaman

# Menggunakan
buku = Buku("Python 101", "Ari", 300)
print(buku)         # Python 101 oleh Ari
print(len(buku))    # 300
