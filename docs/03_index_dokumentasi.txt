════════════════════════════════════════════════════════════════════

1. JANGAN SKIP TEORI
   → Teori adalah fondasi
   → Analogi membantu pemahaman
   → Tanpa teori = hanya hafal syntax

2. BUAT CATATAN SENDIRI
   → Tulis ulang dengan kata sendiri
   → Buat diagram sendiri
   → Ajarkan ke orang lain

   → Jangan takut membuat error
   → Baca pesan error dengan teliti
   → Debug dengan print()

4. PRAKTIK KONSISTEN
   → Minimal 1 jam setiap hari
   → Lebih baik 30 menit setiap hari drpd 5 jam seminggu
   → Consistency > Intensity

5. JANGAN COPY-PASTE
   → Ketik semua kode sendiri
   → Pahami setiap baris
   → Baru then ngerti
